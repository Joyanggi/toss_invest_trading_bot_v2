이 저장소는 사용자 Joyanggi와 Codex, Claude가 함께 개발합니다.

- 원격 저장소: https://github.com/Joyanggi/toss_invest_trading_bot_v2.git
- 기본 브랜치: `main`
- 로컬 작업 경로: `/Users/joyanggi/Documents/toss_invest_trading_bot_v2`

## 0. V1과 완전히 분리한다

- 기존 `/Users/joyanggi/Documents/toss_invest`와 `toss_invest_trading_bot`은 폐기된 V1 참고본이다.
- V1의 앱 코드, 설정, 전략 점수, 상태 파일, 주문 이력, 계좌 데이터, 환경변수 또는 배포 구성을 V2에 복사하지 않는다.
- 과거 연구 결과를 가져와야 할 때는 출처와 한계를 문서화하고, V2 코드로 독립 검증한다.
- 모든 새 작업, 커밋, 푸시는 이 V2 저장소에서 수행한다.

## 1. 작업 전후 WORKLOG를 사용한다

- 작업 전에 `WORKLOG.md`를 먼저 읽는다.
- 기능 추가, 버그 수정, 리팩터링, 연구 결과 변경 후 아래 형식으로 기록한다.

```text
### YYYY-MM-DD - 작성자: Codex | Claude(모델명)
- 문제/목적:
- 진단(근거):
- 수정 내용:
- 건드린 파일/함수:
- 검증 방법과 결과:
- 가정/주의(다른 에이전트가 검토해야 할 점):
```

## 2. 논리적 작업 단위마다 커밋하고 푸시한다

- 완성된 작업은 `origin/main`에 커밋하고 푸시한다.
- 미완성 실험은 푸시하지 않는다.
- Codex 커밋에는 `Co-Authored-By: Codex <codex@openai.com>` 트레일러를 넣는다.
- Claude 커밋에는 해당 Claude 모델의 트레일러를 넣는다.
- 커밋 전에 `git status`와 staged diff를 확인한다.

## 3. 민감정보와 운영 데이터를 커밋하지 않는다

- `.env`, API 키, 시크릿, 계좌 식별자, 주문 원문, 체결 원문을 커밋하지 않는다.
- `data/`, 로컬 캐시, 백업, 포렌식 출력은 Git에서 제외한다.
- 비밀값을 예제나 테스트 fixture에 넣지 않는다.
- 이미 노출된 키를 발견하면 코드에서 지우는 것만으로 충분하다고 보지 말고 사용자에게 폐기와 재발급 필요성을 알린다.

## 4. 연구와 실거래를 분리한다

- 현재 단계는 연구 전용이며 실제 주문 코드를 기본으로 추가하지 않는다.
- 백테스트 변경은 룩어헤드 바이어스, 생존편향, 비용, 체결 시점, 데이터 누락을 검토한다.
- 전략 성과를 하나의 점수로 과장하지 않고 원 수치와 구간별 결과를 함께 남긴다.
- 전략 또는 주문 로직을 실거래 단계로 올릴 때는 별도 사용자 결정과 문서화된 검증 관문이 필요하다.
- 향후 실거래는 환경변수와 웹 설정의 이중 잠금을 사용하고, 봇 인스턴스는 하나만 실행한다.

## 5. 교차 검증 가능한 변경을 만든다

- 판단의 근거, 가정, 행동 변화, 남은 한계를 WORKLOG와 테스트에 남긴다.
- 다른 에이전트의 결과를 그대로 신뢰하지 않고 원자료, 코드 경로, 재현 명령을 확인한다.
- 전략 결과가 좋아질수록 데이터 누수와 과최적화를 먼저 의심한다.
